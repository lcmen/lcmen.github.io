Hotwire vs HTMX vs Unpoly

November 28, 2024 · 8 min read

JavaScript HTML

In this article, I compare three popular libraries for building interactive server-driven UIs — Hotwire, HTMX, and Unpoly — in the context of a simple TODO application I recently developed.

As an engineer with an interest in understanding the underlying mechanisms, I grew tired of reading various discussions on Reddit and HackerNews. I decided to roll up my sleeves and build a simple TODO application to compare these libraries based on typical web application requirements.

The source code for the demo application is available on GitHub, and the application can be viewed in the browser. Please note it's hosted on a free tier, so it may take ~50 seconds to boot on the first request.

Hotwire

Hotwire was developed by 37 Signals to replace Turbolinks with a more modern alternative providing additional capabilities. It comprises three elements:

  • Turbo — Drive, Frames, and Streams for updating page elements
  • Stimulus — a modest JavaScript framework for behaviors
  • Hotwire Native — a framework for building native mobile apps

Turbo supports updating page elements in three ways. Turbo Drive replaces the body of the page without reloading assets. Turbo Frames scope navigation to specific page sections:

HTML
<turbo-frame id="main"> <a href="/">Home</a> <a href="/blog">Blog</a> <a href="/aboutme">About me</a> </turbo-frame>

Turbo Stream updates a part of the page by performing a specified action (e.g., append, remove, replace). It can be delivered as a response to a regular request or via WebSocket broadcasts.

HTMX

HTMX provides a straightforward approach to enhancing HTML with minimal JavaScript, making it ideal for developers who prioritize simplicity and explicit behavior. It extends HTML with attributes that trigger AJAX requests and swap content directly.

Unpoly

Unpoly offers a more declarative approach with comprehensive conventions, minimizing the need for custom tweaks in common use cases.

If we could compare HTMX and Unpoly to backend frameworks, HTMX would be Django then Unpoly would be Ruby on Rails. Unpoly is also the only player from our trio that provides migration scripts to help with upgrading.

Conclusion

While each of these libraries enables the creation of responsive and interactive UIs, they do so in distinct ways:

  • Hotwire is particularly well-suited for seamless Rails integration, real-time WebSocket updates, or building mobile applications.
  • HTMX is ideal for developers who prioritize simplicity and explicit behavior with minimal JavaScript.
  • Unpoly with its comprehensive feature set and conventions, offers a more declarative approach for common use cases.
